# System Logs (Security Perspective)

## Introduction

System logs are records generated by the operating system, applications, and security components to document system activity.

They provide visibility into authentication events, process execution, system changes, network activity, and security incidents.

From a security perspective:

> Logs are one of the most valuable sources of evidence during detection, incident response, and forensic investigations.

---

# Why System Logs Matter

System logs allow administrators and security teams to:

- Monitor system activity
- Detect malicious behavior
- Investigate security incidents
- Troubleshoot system problems
- Maintain compliance
- Reconstruct attack timelines

Without logging, detecting an intrusion becomes significantly more difficult.

---

# Common Log Sources

Operating systems generate logs from multiple components, including:

- Authentication systems
- Kernel events
- Running services
- Network activity
- Applications
- Security software
- Scheduled tasks
- System configuration changes

Each source provides a different view of system behavior.

---

# Linux Logging

Linux systems commonly store logs under:

```
/var/log/
```

Important log files include:

| Log File | Purpose |
|----------|---------|
| `/var/log/auth.log` | Authentication events |
| `/var/log/syslog` | General system activity |
| `/var/log/messages` | System messages (distribution dependent) |
| `/var/log/kern.log` | Kernel events |
| `/var/log/secure` | Security-related events (RHEL-based systems) |
| `/var/log/apache2/` | Apache logs |
| `/var/log/nginx/` | Nginx logs |

Modern Linux distributions often use:

- systemd-journald
- journalctl

---

# Windows Logging

Windows stores logs using the Windows Event Log system.

Common log categories include:

| Log | Purpose |
|------|---------|
| Security | Authentication and security events |
| System | Operating system events |
| Application | Application events |
| Setup | Installation events |
| Forwarded Events | Centralized event collection |

Logs are typically viewed using:

- Event Viewer
- PowerShell
- Windows Event Forwarding (WEF)

---

# Log Severity Levels

Many logging systems classify events by severity.

Common levels include:

- Debug
- Information
- Notice
- Warning
- Error
- Critical
- Alert
- Emergency

Higher severity levels often require immediate investigation.

---

# Why Logs Matter in Security

Logs provide visibility into:

- Successful logins
- Failed authentication attempts
- Privilege escalation
- Process execution
- Service installation
- Account creation
- File access
- Network connections

Logs frequently reveal attacker behavior long before malware is detected.

---

# Common Security Risks

## Log Deletion

Attackers delete logs to remove evidence.

---

## Log Tampering

Modifying log entries to hide malicious activity.

---

## Logging Disabled

Disabling logging prevents security teams from observing attacks.

---

## Sensitive Information in Logs

Improper logging may expose:

- Passwords
- API keys
- Session tokens
- Database credentials
- Personal information

Logs themselves can become valuable attack targets.

---

# Log Rotation

Operating systems periodically archive or remove old log files.

Benefits include:

- Conserving disk space
- Improving performance
- Maintaining long-term log organization

Improper log rotation may result in loss of valuable forensic evidence.

---

# Centralized Logging

Large organizations often forward logs to centralized platforms.

Examples include:

- SIEM solutions
- Log management servers
- Security monitoring platforms

Benefits include:

- Correlation across systems
- Long-term retention
- Faster incident detection
- Threat hunting

---

# Attacker Perspective

Attackers analyze logs to determine:

- Whether their actions were recorded
- Which security controls are active
- How much visibility defenders have
- Whether security monitoring is centralized

They frequently attempt to:

- Clear logs
- Disable logging
- Blend into legitimate activity
- Generate misleading events
- Avoid detection altogether

---

# Defensive Perspective

Security teams use logs to:

- Detect intrusions
- Investigate incidents
- Build attack timelines
- Perform forensic analysis
- Identify compromised accounts
- Monitor privileged activity

Logs are often the primary source of evidence after a compromise.

---

# Anti-Forensics

Sophisticated attackers may employ anti-forensic techniques such as:

- Clearing event logs
- Modifying timestamps
- Disabling audit policies
- Stopping logging services
- Deleting historical log files

Detecting anti-forensic activity is an important aspect of incident response.

---

# Key Takeaways

- System logs record operating system and application activity.
- Logs are critical for detection, monitoring, and forensic investigations.
- Poor logging significantly reduces defensive visibility.
- Attackers frequently attempt to manipulate or disable logging.
- Effective security depends not only on collecting logs but also on monitoring and protecting them.
